# Book Tracker

**Автор:** [Осокина Елизавета Алексеевна]

## Описание

Book Tracker — десктопное приложение для учёта прочитанных книг с возможностью фильтрации и сохранения данных. Программа позволяет:


* добавлять книги с указанием названия, автора, жанра и количества страниц;
* просматривать список всех книг в табличном виде;
* фильтровать книги по жанру и количеству страниц;
* сохранять данные между сеансами работы;
* проверять корректность вводимых данных.

## Функциональность

**Основные возможности:**

1. **Добавление книг:**
   * поле «Название книги» — обязательное для заполнения;
   * поле «Автор» — обязательное для заполнения;
   * выпадающий список «Жанр» с предустановленными вариантами;
   * поле «Количество страниц» — должно быть положительным числом;
   * кнопка «Добавить книгу» — добавляет запись в таблицу.

2. **Фильтрация данных:**
   * по жанру (выбор из списка жанров);
   * по количеству страниц (>100, >200, >300, >500);
   * возможность комбинировать фильтры;
   * кнопка «Сбросить фильтр» — возвращает полный список.

3. **Управление данными:**
   * автоматическое сохранение при добавлении книги;
   * загрузка данных при запуске приложения;
   * отображение в таблице: название, автор, жанр, количество страниц.

4. **Валидация данных:**
   * проверка на пустые поля;
   * проверка, что количество страниц — положительное число.

### Примеры использования

## Пример 1: Добавление книги

Заполните все поля формы:
Название книги: «Мастер и Маргарита»;
Автор: «Михаил Булгаков»;
Жанр: выберите «Роман»;
Количество страниц: 480.
Нажмите кнопку «Добавить книгу».
Книга появится в таблице внизу окна.
Проверьте, что книга сохранилась в файле books_data.json.

## Пример 2: Фильтрация по жанру

В панели фильтров выберите жанр «Фантастика».
Нажмите «Применить фильтр».
В таблице останутся только книги жанра «Фантастика».
Нажмите «Сбросить фильтр», чтобы увидеть все книги.
Пример 3: Фильтрация по страницам
В панели фильтров выберите «>200».
Нажмите «Применить фильтр».
В таблице будут книги с количеством страниц больше 200.

## Пример 4: Комбинированная фильтрация

Выберите жанр «Роман» и критерий «>300».
Нажмите «Применить фильтр».
В таблице появятся только романы с количеством страниц больше 300.
Нажмите «Сбросить фильтр», чтобы вернуться к полному списку.

### Тестирование
##Тест 1: Корректное добавление

Данные: название — «1984», автор — «Джордж Оруэлл», жанр — «Антиутопия», страниц — 328.
Ожидаемый результат: книга добавлена в таблицу и сохранена в JSON.
Проверка: откройте файл books_data.json — книга должна быть в списке.

## Тест 2: Пустые поля

Данные: все поля пустые или пропущено название/автор/жанр.
Ожидаемый результат: сообщение об ошибке «Все поля должны быть заполнены!».
Проверка: книга не добавлена в таблицу, история не обновлена.

## Тест 3: Некорректное количество страниц

Данные: страниц — «abc», «−50», «0» или пустое поле.
Ожидаемый результат: сообщение об ошибке «Количество страниц должно быть положительным числом!».
Проверка: книга не добавлена, таблица не изменилась.

## Тест 4: Фильтрация

Действия: добавьте 5–7 книг разных жанров и с разным количеством страниц. Примените разные фильтры.
Ожидаемый результат: фильтры работают корректно, показывают только соответствующие книги.
Проверка: комбинируйте фильтры — результат должен соответствовать обоим критериям.

## Тест 5: Сохранение данных

Действия: добавьте 2–3 книги, закройте приложение (через крестик окна), запустите заново.
Ожидаемый результат: все книги отображаются в таблице.
Проверка: сравните данные в таблице с содержимым books_data.json.

### Технические детали
Язык программирования: Python 3;
Графический интерфейс: Tkinter (стандартная библиотека Python);
Хранение данных: JSON‑файл (books_data.json);
Валидация: проверка пустых полей и числового значения страниц;
Кодировка: UTF‑8 (поддержка кириллицы);
Формат данных: структурированный JSON с отступами для удобства чтения.
